# AI案件作成フロー 初回オンボーディングPOPUP プロトタイプ

## 📋 概要

AI案件作成フローの入口である「1文入力画面」において、ユーザーの離脱率が高い（遷移率 約47%）という課題を解決するため、初回訪問時に**「作業は簡単であること」「AIが代行してくれること」**を視覚的に伝えるオンボーディングPOPUPのプロトタイプです。

## 🎯 目的

- 心理的ハードルを下げる
- 次画面への遷移率（入力完了率）を向上させる
- 「難しそう・面倒くさそう」という認知負荷を下げる

## 🎨 デザインコンセプト

### 訴求軸：手軽さ・簡単さ（Pattern A）

- **メッセージ**: 「難しそう・面倒くさそう」という認知負荷を下げることを最優先
- **キーワード**: 「一言入力するだけ」「AIにおまかせ」「3分で完了」
- **トンマナ**: 既存サイトのトンマナを守りつつ、AI機能らしい「先進性」と、アシスタントとしての「親しみやすさ・安心感」を両立

## 📱 対応デバイス

- **PC版**: 3ステップを横並び（3カラム）で配置
- **SP版**: 3ステップを縦積み（リスト形式）で配置（レスポンシブ対応）
- **重要**: SP版はファーストビューでの視認性を重視し、スクロールなしでCTAボタンまで収まる密度感を意識

## 🎬 UI構成

### 表示形式
- **モーダル（POPUP）**
- **背景**: 黒の半透明オーバーレイ（Opacity 50%）
- **閉じる方法**: 
  - 右上の「×」アイコン
  - 背景タップ
  - CTAボタン押下
  - ESCキー

### 構成要素

#### ヘッダー
- **タイトル**: 面倒な入力はAIにおまかせ！
- **サブタイトル**: 3ステップで、最短3分で募集開始できます。

#### ボディ（3ステップ）

**Step 1: 一言入力するだけ**
- 説明: 「ロゴを作りたい」など、思いつきの言葉でOK！
- アイコン: 💬 吹き出し・入力

**Step 2: AIがまるっと作成**
- 説明: 面倒な要件定義や文章作成はAIが代行します。
- アイコン: ✨ 魔法の杖・ロボット

**Step 3: 待つだけで完成**
- 説明: 最適な内容に仕上がります。修正も可能です。
- アイコン: ⏱️ 時計・完成

#### フッター
- **CTAボタン**: 一言入力してみる

## 🚀 使い方

### ローカルで実行

1. リポジトリをクローン
```bash
git clone https://github.com/asuka-kawaguchi-coconala/ai-job-onboarding-popup-prototype.git
cd ai-job-onboarding-popup-prototype
```

2. ブラウザで開く
```bash
# 方法1: 直接ファイルを開く
open index.html

# 方法2: ローカルサーバーを起動（推奨）
python -m http.server 8000
# または
npx serve
```

3. ブラウザで `http://localhost:8000` にアクセス

### モーダルの表示

- 「オンボーディングPOPUPを表示」ボタンをクリック
- 実際の実装では、初回訪問時に自動表示（現在はコメントアウト）

## 🔧 カスタマイズ

### 初回訪問時の自動表示

`script.js` の以下の部分を有効化してください：

```javascript
window.addEventListener('load', () => {
    const isFirstVisit = !localStorage.getItem('aiJobOnboardingShown');
    if (isFirstVisit) {
        setTimeout(() => {
            openModal();
        }, 500);
    }
});
```

### オーバーレイの透明度

`styles.css` の以下の部分を変更：

```css
.modal-overlay {
    background-color: rgba(0, 0, 0, 0.5); /* 0.4-0.6の範囲で調整可能 */
}
```
